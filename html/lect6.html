Lecture 6: Forms and Multimedia

Forms

  • <form>: Container for form elements with attributes like enctype for file uploads
  • <input>: New types like date, email, file, range, search, tel, url
  • <datalist>: Provides autocomplete options for text inputs
  • <optgroup>: Groups related options in dropdowns
  • <output>: Displays calculation results
  • <meter>: Shows scalar measurement within a range

Advanced Form Attributes

  • multiple: Allows multiple file/email selections
  • formaction: Overrides form action for specific buttons
  • formenctype: Specifies encoding for file uploads
  • min and max: For number/date inputs
  • HTML5 validation: novalidate, formnovalidate

Form Submission Methods

  • GET: Visible in browser history and bookmarks
  • POST: Can handle file uploads and large data
  • Consider PUT or DELETE for REST APIs

Form Best Practices

  • Use semantic input types (email, tel) for mobile keyboards
  • Implement client-side validation with JavaScript
  • Include proper error messages and aria-labels
  • Optimize form layout for mobile responsiveness

Advanced Audio Features

  • JavaScript API for custom controls
  • crossorigin attribute for CORS requests
  • preload to optimize loading behavior
  • Web Audio API for advanced processing

Video Enhancements

  • playsinline for mobile browsers
  • muted required for autoplay on many browsers
  • Subtitles with <track> element
  • Picture-in-picture API support

Modern Multimedia Features

  • Media Fragments: Deep linking to specific times
  • Adaptive Streaming: HLS or DASH for quality adjustment
  • WebRTC: Real-time communication in browsers
